# Graph Packaging Edge-Case Goldset Proof

## Status

Implements `TASK-GRAPH-QUALITY-GOLDSET-002`, the G4b measurement expansion after G4a saturated the original graph-quality fixture.

## Scope

The graph-quality goldset advances from version `1.0` to `1.1`. The producer in `merger/lenskit/architecture/import_graph.py` is unchanged. The expanded fixture and evaluator add explicit cases for:

- package `__init__.py` resolution;
- a conventional `src/` import root;
- a namespace-style package path without `__init__.py`;
- ambiguous search roots with two possible local targets;
- invalid Python syntax;
- overlapping path-layer precedence.

## Deterministic semantics

Local-resolution cases still ask whether an exact `file:source -> file:target` edge exists. The two new import-root cases are allowed to miss and are recorded individually rather than hidden behind a perfect aggregate score.

External-preservation cases may now declare `forbidden_targets`. A case passes only when the external module edge exists and no edge points to any forbidden local candidate. This makes ambiguity preservation falsifiable.

Parse-failure cases pass only when the declared invalid file has no emitted file node. The report also records graph coverage counts so the skipped node can be reconciled with `files_seen` and `files_parsed`.

## Reproduced baseline

The baseline is generated by the evaluator against the committed fixture:

- local resolution: `5/7`, recall `0.714286`;
- external preservation: `3/3`, accuracy `1.0`;
- layer assignment: `8/8`, accuracy `1.0`;
- unknown file-layer share: `10/17`, or `0.588235`;
- declared parse-failure handling: `1/1`;
- coverage: `18` Python files seen, `17` parsed, one parse failure.

The two local misses are exactly:

- `src-layout-import-root-gap`;
- `namespace-import-root-gap`.

The lower aggregate recall is caused by new denominator cases, not a producer change. Existing G4a cases remain green.

## Decision

G4b does not add `src/` or namespace heuristics. Directory-name guessing would convert an observed gap into unsupported certainty. A later producer slice needs an explicit source-root contract or packaging-metadata policy before resolving these cases.

## Verification surface

- goldset loader validation covers version `1.1`, duplicate IDs, parse cases, and malformed forbidden-target lists;
- fixture-path checks include local targets, forbidden candidates, layer paths, and parse failures;
- the committed baseline is regenerated from the real producer and compared byte-semantically through parsed JSON;
- focused Graph Quality Goldset and Architecture Import Graph tests run in CI;
- Ruff and structural diff checks remain blocking.

## Non-claims

This work does not establish effective `sys.path`, installed-package state, Python runtime import order, namespace-package completeness, graph completeness, runtime causality, retrieval benefit, change impact, test sufficiency, or readiness to enable graph ranking by default.
